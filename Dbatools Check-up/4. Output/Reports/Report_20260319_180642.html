📘 SQL Health Report - User Guide

🎯 Understanding the Report

This report provides a comprehensive health assessment of your SQL Server instances using read-only checks organized into categories like Storage, Security, Performance, and Maintenance.

📊 Health Scoring

Each instance receives a health score (0-100%) based on weighted checks:

  • Pass (Green): Check passed - no action needed (100 points)
  • Attention (Amber): Warning condition - review recommended (60 points)
  • Fail (Red): Critical issue - immediate action required (0 points)
  • Info (Blue): Informational only - does not affect score
Tip: The overall health score is calculated by averaging weighted check results. High-priority checks (like security issues) have more weight than low-priority checks.

🗂️ Navigating the Report

Overview Tab: See all instances at a glance with category breakdowns

  • Click category cards to filter findings by that category
  • Use "Hide above X%" to focus on problem instances
  • Click any instance name to view detailed findings

Instance Tab: Deep dive into a single instance

  • Click category chips to filter findings by category
  • Click KPI pills (Pass/Attn/Fail/Info) to filter by status
  • Use the dropdown to switch between instances

All Findings Tab: View findings across all instances

  • Quick filters: Jump to common views (failures only, attention+failures, etc.)
  • Advanced filters: Filter by category, spoke, priority, status, label, or function
  • Hide toggles: Remove Pass/Info/Entry checks from view

🔍 Filtering & Searching

All tables support sorting by clicking column headers. The arrow (^ or v) shows sort direction.

Use the filter bars to narrow results:

  • Category: Group checks by domain (Storage, Security, etc.)
  • Spoke: Filter by the PowerShell script that ran the check
  • Priority: High/Medium/Low importance level
  • Status: Pass/Attention/Fail/Info
  • Label: Search check names (e.g., "backup", "memory")
  • Function: Filter by dbatools function (e.g., "Get-Dba*")
Pro Tip: Use "Hide: Entries" to remove granular per-database or per-job findings and focus on instance-level checks. Entry checks have an ENTRY tag.

💾 Import & Export

Import JSON: Load findings from a different health check run

Export JSON: Save the current findings for archival or comparison

Export Workbook: Download an Excel file with two sheets:

  • Instances Overview: Health scores by category for all instances
  • All Findings: Complete findings list with all details

📋 Entry Checks

Some checks return multiple results (e.g., one per database or per job). These are marked with an ENTRY tag. Use the "Hide: Entries" toggle to collapse these into a summary view when you want a high-level overview.

🎨 Understanding Colors

The color scale is continuous from red to green:

  • 0-40%: Dark red - critical
  • 40-50%: Red - poor
  • 50-80%: Amber - warning
  • 80-90%: Light green - good
  • 90-100%: Dark green - excellent

🔗 URL Hash Navigation

The report preserves your view in the URL. You can bookmark or share links to specific:

  • Tabs (e.g., #tab=overview)
  • Instances (e.g., #tab=instance&instance=SQLPROD-01)
  • Filtered views (e.g., #tab=allfindings&status=fail&cat=Security)

🖨️ Printing

Use your browser's print function (Ctrl+P / Cmd+P) to generate a PDF. Buttons and interactive elements are hidden for clean printouts.

⚙️ Suite Version

The suite version is displayed in the top-right corner. This helps identify which version of the SQL Health Suite generated this report when troubleshooting or comparing results.
